üéØ CSS Specificity Priority

Rule: Inline CSS > Internal CSS > External CSS
However, specificity of selectors can override this rule!
Priority Level Selector Type Weight Example
1 (Highest) Inline Style 1000 style="color: blue;"
2 ID Selector 100 #sr { }
3 Class/Pseudo-class/Attribute 10 .jr { }
4 (Lowest) Element/Pseudo-element 1 p { }

‚ö†Ô∏è Troubleshooting Guide

Problem 1: CSS Not Loading

  • Check if style.css is in the same directory as index.html
  • Verify the path in link tag: <link rel="stylesheet" href="style.css">
  • Clear browser cache (Ctrl+F5 or Cmd+Shift+R)
  • Check browser console for 404 errors

Problem 2: Image Not Displaying

  • Ensure pics folder exists in project directory
  • Check image filename matches exactly: pic.jpg
  • Verify image path: src="pics/pic.jpg"
  • Check file extension (jpg vs jpeg vs png)

Problem 3: Styles Not Applying as Expected

  • Check CSS selector specificity (ID > Class > Element)
  • Uncomment internal styles in HTML to test priority
  • Use browser DevTools (F12) to inspect applied styles
  • Look for typos in class/ID names (case-sensitive!)
  • Verify CSS syntax (missing semicolons, brackets)

Problem 4: font-weight Not Working

  • font-weight: 10px; is INVALID
  • Use: font-weight: normal; (400)
  • Or: font-weight: bold; (700)
  • Or: font-weight: 100-900; (numeric values)

‚úÖ Best Practices

  1. Avoid Inline Styles: Keep HTML and CSS separated for maintainability
  2. Use External CSS: Easier to manage and cache
  3. Consistent Naming: Use meaningful class/ID names
  4. Comment Your Code: Explain complex selectors
  5. Use Classes Over IDs: IDs should be unique, classes are reusable
  6. Valid CSS Values: Check property-value compatibility
  7. Organize Your CSS: Group related styles together
